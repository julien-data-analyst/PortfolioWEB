SAÉ 5.VCOD.01 : Analyse et conception d'outil décisionnel


  • A. Présentation du projet

    Ce projet avait pour objectif de créer une application web sur le thème du cinéma. Les utilisateurs pouvaient faire des recherches de films selon divers critères. Pour y parvenir, nous avons utilisé le framework Flask en Python pour la conception de l'application web, ainsi qu'une base de données SQLite afin de gérer l'interaction avec le site.

  • B. Organisation du travail

    • Modalité : travail de groupe (4)
    • Suivi : réunion présentielle/visioconférence
    • Ressources : cours de développement web (Flask), modélisation logiciel (UML, DC, DCU)
    • Outil : Python (Flask), langages web (HTML, CSS, JS) et GitHub
    • Livrables : présentation orale (10 minutes) et rendre l'arborescence du projet

  • C. Démarche et résolution de problème

    Ce projet s'est articulé en plusieurs étapes clés :

    • Conception des diagrammes UML : Cette phase a permis, d'une part, de définir les principales fonctionnalités de l'application en adoptant une approche centrée sur l'utilisateur, et d'autre part, d'établir un modèle structuré pour la base de données.
    • Préparation et insertion des données : Nous avons travaillé à partir d'un fichier CSV contenant 44 691 films, incluant leur synopsis, leur réalisateur, leurs acteurs et d'autres informations. Cette étape a été la plus chronophage en raison des nombreuses erreurs présentes dans les données, nécessitant un nettoyage minutieux avant leur insertion.
    • Requêtes sur la base de données : Mon rôle consistait à interroger la base de données pour générer des visuels graphiques, des indicateurs de performance (KPI) et des formulaires. Cette tâche s'est avérée complexe, car notre modèle relationnel manquait de stabilité, nous obligeant à ajuster régulièrement le schéma de la base.
    • Mise en page du site web : Cette étape concernait la structuration et le design de la page web à l'aide des langages HTML, CSS et JavaScript. L'objectif était de créer une interface à la fois attrayante et intuitive pour les utilisateurs.
    • Mise en place des formulaires : Avec un camarade, nous avons développé deux formulaires interactifs. Le premier permettait de rechercher des films par titre ou par genre, tandis que le second facilitait la recherche de films en fonction du réalisateur.

    Afin de mieux coordonner notre travail et de fluidifier nos échanges, nous avons utilisé GitHub, un gestionnaire de projets. Cet outil nous a permis de collaborer efficacement sur chaque partie du projet sans être contraints d'attendre les avancées des autres membres de l'équipe.

  • D. Conclusion

    Pour conclure, ce projet m'a permis de découvrir le contexte de création d'une application web passant de la conception au lancement du site après finition. Cependant, je dois apprendre à mieux travailler en équipe pour améliorer la performance du développement et ne pas avoir un manque de communication.
