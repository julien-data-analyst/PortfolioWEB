SAÉ 5.02 : Migration de données vers ou depuis un environnement NoSQL


  • A. Présentation du projet

    Un ensemble de données concernant Olist, le plus grand site e-commerce brésilien, a été mise à disposition sur Kaggle. Elle permettait de travailler sur les commandes, les avis des clients, les paiements, les produits et les vendeurs. L'objectif de ce projet était de créer une base de données via MongoDB à partir de cet ensemble de données (à l'exception de la géolocalisation).

  • B. Organisation du travail

    • Modalité : travail individuel
    • Ressources : cours environnement NoSQL (Neo4j, MongoDB, Redis)
    • Outil : Python (Notebook, pymongo), MongoDB Compact
    • Livrables : jupyter-notebook, rapport (latex)

  • C. Démarche et résolution de problème

    Le projet a débuté par une phase d'importation et de préparation des données sous Python. Cette étape était essentielle pour analyser la structure de chaque jeu de données et les ajuster si nécessaire. Cependant, elle s'est révélée complexe, car il fallait comprendre en détail l'organisation de chaque dataset. Ensuite, j'ai travaillé sur l'intégration de ces données. Cela se faisait par la librairie pymongo qui nous a permis d'effectuer différentes requêtes. Chaque fichier importé sous Python était ensuite stocké dans une collection de la base. Enfin, j'ai exploité la puissance de MongoDB pour analyser plusieurs axes, notamment :

    • Le prix moyen des articles par catégorie
    • Les vendeurs les mieux notés
    • Les 7 États les plus prolifiques
    • Les 7 États avec le délai de livraison le plus court
    • La catégorie de produit la mieux notée

    Ces analyses nécessitaient une bonne compréhension du schéma de la base de données, afin d'identifier les colonnes pertinentes et les relations entre les différentes collections. Pour optimiser les performances des requêtes, j'ai mis en place des index sous MongoDB, réduisant ainsi le temps d'exécution de certaines requêtes de 1 heure à seulement 10 secondes.

  • D. Conclusion

    Pour conclure, ce projet m'a permis de découvrir le SGBD MongoDB et ses avantages. J'ai appris à faire comme sur SQL, la création de requêtes pour de l'analyse exploratoire. Cependant, il faudra que je m'améliore au niveau des axes d'analyses et des visuels à utiliser.
